# Contributing to Awesome Agent Skills Security

Thank you for your interest in contributing!

## Guidelines

### Adding a Resource

1. **Check it's not already listed** — search the README first.
2. **Pick the right category** — if none fits, propose a new one.
3. **Use the standard format:**
   - Papers: `- 📄 **[Title](URL)** — Authors, Venue Year. One-sentence description.`
   - Tools: Add to the table with GitHub badge.
   - Blog posts: `- 🔗 **[Title](URL)** — Source, Year. One-sentence description.`
4. **Keep descriptions concise** — one sentence, focus on what's unique.

### Quality Standards

- **Papers**: Must be peer-reviewed or on arXiv. Preprints are fine if technically sound.
- **Tools**: Must be open-source with commits in the last 6 months.
- **Blog posts**: Must provide original technical analysis, not just opinions.
- **Standards/Frameworks**: Must be from recognized organizations.

### What We're Looking For

- Novel attack vectors on agent tool use
- Defense mechanisms for skill ecosystems
- Benchmarks for evaluating agent security
- Real-world incident analyses
- Formal methods applied to agent safety

### What We're NOT Looking For

- General LLM safety papers without agent/tool connection
- Marketing materials or product announcements
- Papers without public access (no paywalled-only papers)
- Duplicate entries

### Process

1. Fork → Branch → Edit → PR
2. One resource per PR (or batch related additions)
3. Include a brief justification in the PR description
4. Maintainers will review within 1 week

## Code of Conduct

Be respectful and constructive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
